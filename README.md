1.系统架构设计
A.规划策略层（Planner Agent - 长链推理）
任务拆解：接收模糊指令（如“测试商城购买仓库”），利用长链推理（Chain-of-Thought）将任务拆解为：登录 -> 余额检查 -> 商品选择 -> 支付验证 -> 背包检查。
边界边界：自动推导边界条件（如：余额不足、网络断开），生成覆盖异常路径的测试计划。
B. 动作执行层（Executor Agent - 环境交互）
多模式采集：通过截图/UI树解析当前状态。如果是游戏测试，会采集YOLO/OpenCV识别实体。
动作回放：将高层指令（“购买最贵的道具”）映射为高级操作坐标或API调用，并实时采集反馈日志。
C. 质量层审计（Auditor Agent - 反思与验证）
闭环反思：对比“执行日志”与“规划预期”。若发现不一致，Agent会进行“自我诊断”：是脚本执行失败，还是产品逻辑Bug。
报告聚合：不仅输出Pass/Fail，更会输出包含Bug等级、复现步骤及初步修复建议的重构报告。

3、项目解决的核心痛点
黑盒测试的“意图断层”：传统的脚本测试（如 Selenium/Appium）无法处理复杂的 UI 动作或随机的游戏场景。本系统通过视觉模型和语义理解，让 Agent 像人类玩家一样理解“为什么要点这个按钮”。
异常捕获的“非重构”问题：游戏崩溃或逻辑错误往往隐藏在海量日志中。系统利用LLM的审计能力，能从杂乱的日志中精准找出Bug触发路径和根源，解决“测出问题但复现难”的痛点。
维持成本高：产品UI一改，剧本全废。该系统基于自然语言描述生成动态动作流，具有极高的增益能力，大幅降低了剧本维持成本。
